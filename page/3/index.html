1. зӮ№еҮ»дәӢд»¶й—®йўҳ

з§»еҠЁз«Ҝ click

з”ұдәҺжүӢжңәз«ҜжңүдәӣжөҸи§ҲеҷЁдёҚеӯҳеңЁ click дәӢд»¶пјҢеҸҜд»ҘеңЁе…ЁеұҖиҝҷж ·еӨ„зҗҶ

и§ЈеҶіж–№жі•1(дәІжөӢжңүж•Ҳ)пјҡ

еҸҜд»Ҙи§„йҒҝеҪ“clickе’ҢtouchstartеҗҢж—¶еӯҳеңЁж—¶зӮ№еҮ»дёҖж¬ЎдҪҶи§ҰеҸ‘дёӨж¬Ўй—®йўҳпјҲеҰӮ ipadпјүпјҢжӯӨж–№жі•еҸҜд»Ҙи§ЈеҶіеӨ§йғЁеҲҶй—®йўҳгҖӮ

дәӢд»¶е§”жүҳй—®йўҳпјҡ

жӯӨж®өд»Јз ҒеңЁ ios12 зі»з»ҹдёӢзҡ„ safari е’Ң webview дёӯжҳҫзӨәж— ж•Ҳ

еҺҹеӣ пјҡios12 дёӢеҲ©з”Ё document е’Ң body иҝӣиЎҢдәӢд»¶е§”жүҳз»‘е®ҡ click жҳҜж— ж•Ҳзҡ„гҖӮ

и§ЈеҶіеҠһжі•2(дәІжөӢжңүж•Ҳ)пјҡ

  • е°Ҷ click дәӢд»¶е§”жүҳеҲ°вҖӢвҖӢвҖӢвҖӢвҖӢйқһ document жҲ– body зҡ„вҖӢвҖӢзҲ¶зә§е…ғзҙ дёҠ
  • з»ҷвҖӢзӣ®ж Үе…ғзҙ еҠ дёҖжқЎж ·ејҸи§„еҲҷ cursor: pointer;
  • е°Ҷзӣ®ж ҮвҖӢе…ғзҙ жҚўжҲҗ жҲ–иҖ… button зӯүеҸҜзӮ№еҮ»зҡ„вҖӢе…ғзҙ ;

еҸ‘иЎЁдәҺ 2018-02-06   |   еҲҶзұ»дәҺ Git

git flow

иҜҫд»¶ејҖеҸ‘ git ж ҮеҮҶжөҒзЁӢ

д»Ҙдё»еҲҶж”ҜmasterдёәдҫӢпјҢжҜҸеҪ“ејҖеҸ‘дёҖдёӘж–°еҠҹиғҪж—¶пјҢйңҖиҰҒд»Һmaster checkoutдёҖдёӘж–°зҡ„еҲҶж”ҜпјҢжңҖеҘҪд»ҘеҠҹиғҪиҝӣиЎҢеҢәеҲҶпјҡ

  1. feature/JTZXKJ-xxx
  2. bugfix/JTZXKJ-xxx
  3. improve/JTZXKJ-xxx

дёҖдёӘеҲҶж”Ҝзҡ„д»Јз ҒеҫҲеҸҜиғҪејҖеҸ‘дәҶеҘҪеҮ еӨ©пјҢжҜҸеӨ©йғҪжңүж–°зҡ„жҸҗдәӨпјҢиҝҷдёӘж—¶еҖҷпјҢиҮӘе·ұеҲҶж”Ҝзҡ„д»Јз ҒиҰҒеҗҲе№¶еҲ°masterпјҢеҝ…з„¶дјҡжңүеҫҲеӨҡзҡ„commitпјҢйҖҡеёёжқҘиҜҙдёҖдёӘеҠҹиғҪејҖеҸ‘пјҢbugдҝ®еӨҚпјҢеҸӘйңҖиҰҒдёҖдёӘcommit,жӯӨж—¶пјҢе°ұйңҖиҰҒе°ҶеӨҡдёӘcommitеҗҲе№¶дёәдёҖдёӘгҖӮдёәдәҶзЎ®дҝқеҪ“еүҚиҮӘе·ұзҡ„еҲҶж”Ҝд»Јз ҒзӣёеҜ№дәҺmasterжҳҜжңҖж–°зҡ„пјҢеӣ жӯӨйңҖиҰҒеҒҡдёӢеҸҳеҹәж“ҚдҪңпјҲд»ҘmasterеҒҡеҹәеҮҶпјүиҝҮзЁӢпјҡ

  1. commitеҪ“еүҚеҲҶж”Ҝзҡ„жүҖжңүд»Јз Ғ
  2. еҲҮжҚўеҲ°masterеҲҶж”ҜпјҢgit pull origin master
  3. еҲҮжҚўеӣһиҮӘе·ұзҡ„еҲҶж”Ҝ git checkout feature/JTZXKJ-xxx
  4. git rebase -i master, иҝӣе…Ҙvimзј–иҫ‘зҠ¶жҖҒпјҲMake a list of the commits which are about to be rebasedпјү
  5. дҝ®еӨҚйҷӨз¬¬дёҖдёӘcommitзҡ„pickеӨҙйғЁж Үеҝ—еӨ–пјҢе°Ҷе…¶дҪҷcommitзҡ„еӨҙйғЁж Үеҝ—pickж”№дёәs
  6. йҖҖеҮәvim,и®°еҫ—wqдҝқеӯҳ
  7. git push origin feature/JTZXKJ-xxx
  8. еҰӮжһңеҲҶж”Ҝfeature/JTZXKJ-xxxеңЁиҝңзЁӢд»“еә“е·Із»ҸеӯҳеңЁпјҢеҲҷйңҖиҰҒgit push origin feature/JTZXKJ-xxx -u вҖ”forceгҖӮеҰӮжһңиҝңзЁӢеҲҶж”ҜдёҚеӯҳеңЁпјҡgit push вҖ”set-upstream origin feature/JTZXKJ-xxx
  9. жҸҗдәӨmerge request
  10. д»Јз Ғе®ЎжҹҘпјҢд»Јз ҒеҗҲе№¶пјҢеҰӮжһңд»Јз Ғе®ЎжҹҘиҝҮзЁӢдёӯпјҢи®ӨдёәеҠҹиғҪејҖеҸ‘жңүй—®йўҳпјҢеҲҷйңҖиҰҒйҮҚж–°дҝ®ж”№еҲҶж”Ҝзҡ„д»Јз ҒпјҢдҝ®ж”№е®ҢжҲҗеҗҺпјҢйҮҚеӨҚжӯӨиҝҮзЁӢ(еҲҶж”ҜејәжҺЁж“ҚдҪңпјҢдёҚйңҖиҰҒе…ій—ӯе·Іжңүзҡ„merge reqquest)
  11. 1~4зҡ„ж“ҚдҪңиҝҳеҸҜд»Ҙз®ҖеҢ–дёәgit rebase -i origin master

git еёёз”Ёе‘Ҫд»Ө

  • git add * е’Ң git add .
    git add .дјҡжҸҗдәӨжүҖжңү untracked ж–Үд»¶гҖҒж–Үд»¶еӨ№гҖҒеҢ…жӢ¬д»Ҙ .ејҖеӨҙзҡ„ж–Үд»¶
    git add *дјҡжҸҗдәӨжүҖжңү untracked ж–Үд»¶еӨ№гҖҒж–Үд»¶гҖҒдҪҶдёҚеҢ…жӢ¬.ејҖеӨҙзҡ„ж–Үд»¶
  • git reset HEAD <file> ж’Өеӣһе·Із»Ҹиў« add дҪҶжІЎжңүиў« commit зҡ„ж–Үд»¶
  • git checkout --track origin/special иҜҘе‘Ҫд»ӨдјҡеңЁжң¬ең°еҲӣе»әдёҖдёӘ special еҲҶж”Ҝз”ЁдәҺи·ҹиёӘиҝңз«Ҝзҡ„ special еҲҶж”Ҝе№¶е°Ҷжң¬ең°еҲҶж”ҜеҲҮжҚўеҲ° special еҲҶж”Ҝ
  • git checkout [filename]ж”ҫејғеҜ№жң¬ең°е·Ідҝ®ж”№дҪҶе°ҡжңӘ add зҡ„ж–Үд»¶зҡ„дҝ®ж”№пјҢиҝҳеҺҹе…¶еҲ°жңӘдҝ®ж”№еүҚзҡ„зҠ¶жҖҒгҖӮ
    жӯӨе‘Ҫд»Өе’Ң github client зҡ„ discard changesзҡ„еҚҙеҲ«: discard changes еҸҜд»Ҙж”ҫејғе·Із»Ҹиў« add зҡ„ж–Үд»¶пјҢдҪҶgit checkout [filename]дёҚиғҪпјҢеҸӘиғҪж”ҫејғжІЎжңүиў« add зҡ„
  • git checkout . ж”ҫејғжүҖжңүжң¬ең°е·Ідҝ®ж”№дҪҶе°ҡжңӘ add зҡ„ж–Үд»¶зҡ„дҝ®ж”№пјҢдҪҶдёҚеҢ…жӢ¬ж–°еўһзҡ„ж–Үд»¶
  • git branch -a еҲ—еҮәжң¬ең°е’ҢиҝңзЁӢеҲҶж”Ҝ
  • git branch [name] -D еҲ йҷӨжң¬ең°еҲҶж”Ҝ
  • git commit -m'' вҖ”-amend еҗҲе№¶еӨҡдёӘ commit дёәдёҖдёӘ
  • git reset --soft origin/[branch] ж’ӨеӣһжүҖжңүзҡ„ commit и®°еҪ•пјҢеҗҢжӯҘеҲ° origin/[branch]гҖӮ
    е®ғжң¬иҙЁдёҠжҳҜж’Өй”ҖдәҶдёҠдёҖж¬Ў git commit е‘Ҫд»ӨгҖӮ еҪ“дҪ еңЁиҝҗиЎҢ git commit ж—¶пјҢGit дјҡеҲӣе»әдёҖдёӘж–°зҡ„жҸҗдәӨпјҢе№¶з§»еҠЁ HEAD жүҖжҢҮеҗ‘зҡ„еҲҶж”ҜжқҘдҪҝе…¶жҢҮеҗ‘иҜҘжҸҗдәӨгҖӮ еҪ“дҪ е°Ҷе®ғ reset еӣһ HEAD~пјҲHEAD зҡ„зҲ¶з»“зӮ№пјүж—¶пјҢе…¶е®һе°ұжҳҜжҠҠиҜҘеҲҶж”Ҝз§»еҠЁеӣһеҺҹжқҘзҡ„дҪҚзҪ®пјҢиҖҢдёҚдјҡж”№еҸҳзҙўеј•е’Ңе·ҘдҪңзӣ®еҪ•гҖӮ зҺ°еңЁдҪ еҸҜд»Ҙжӣҙж–°зҙўеј•е№¶еҶҚж¬ЎиҝҗиЎҢ git commit жқҘе®ҢжҲҗ git commit -m'' --amend жүҖиҰҒеҒҡзҡ„дәӢжғ…дәҶ
  • git log жҹҘзңӢж—Ҙеҝ—пјҢйҖҖеҮәfn + Q
  • git log -p [file] вҖ” жҹҘзңӢжҹҗдёӘж–Үд»¶зҡ„еҶ…е®№еҸҳеҢ–иҜҰжғ…
  • git log вҖ”online з®ҖжҙҒжЁЎејҸжҳҫзӨә log и®°еҪ•
  • git fetchпјҡеҪ“жӯӨе‘Ҫд»Өд»ҺжңҚеҠЎеҷЁдёҠжҠ“еҸ–жң¬ең°жІЎжңүзҡ„ж•°жҚ®ж—¶пјҢе®ғе№¶дёҚдјҡдҝ®ж”№е·ҘдҪңзӣ®еҪ•дёӯзҡ„еҶ…е®№гҖӮ е®ғеҸӘдјҡиҺ·еҸ–ж•°жҚ®з„¶еҗҺи®©дҪ иҮӘе·ұеҗҲе№¶гҖӮ з„¶иҖҢпјҢжңүдёҖдёӘе‘Ҫд»ӨеҸ«дҪң git pull еңЁеӨ§еӨҡж•°жғ…еҶөдёӢе®ғзҡ„еҗ«д№үжҳҜдёҖдёӘ git fetch зҙ§жҺҘзқҖдёҖдёӘ git merge е‘Ҫд»ӨгҖӮgit pull = git fetch + git merge
  • git еӣһж»ҡ
  1. git -log жүҫеҲ°еҜ№еә”зҡ„
    commit 1a864433ec4a9bd712d921859a8a474510e7e871
  2. git revert [commitId]: еҪ“жҸҗдәӨиҝҮзҡ„жҹҗдёҖ commit дёҚи®әеӨҡд№…пјҢйғҪеҸҜд»ҘдҪҝз”ЁжӯӨе‘Ҫд»ӨйҖҶеҗ‘жҒўеӨҚеҪ“ж—¶зҡ„ commit зҡ„жӣҙж”№еҶ…е®№е№¶з”ҹжҲҗдёҖдёӘж–°зҡ„ commit пјҢзӣёеҪ“дәҺеӣҫеҪўеҢ–зҡ„ revert (:q йҖҖеҮәж—¶дјҡиҮӘеҠЁ commit )пјҢд№ҹеҸҜд»ҘејәеҲ¶е…ій—ӯйҮҚж–°дҝ®ж”№д»Јз Ғз„¶еҗҺжүӢеҠЁж·»еҠ  commitпјҲжҺЁиҚҗдҪҝз”ЁпјҒпјҒпјҒпјү
  3. git reset --soft [commitId] дјҡе°ҶжҹҗдёӘ commit д№ӢеүҚзҡ„жүҖжңүжӣҙж”№еҶ…е®№жҡӮеӯҳеҲ°жң¬ең°пјҢдёҚдјҡдёўеӨұпјҢдҪҶжҳҜдјҡе°ҶеӨҡдёӘ commit зҡ„жӣҙж”№ж··ж·Ҷ
  4. git reset вҖ”-hard [commitId] дјҡжҢҮеҗ‘жҹҗдёӘ commit е№¶е°Ҷд№ӢеүҚзҡ„еҶ…е®№е…ЁйғЁиҲҚејғпјҢжң¬ең°д№ҹдёҚдјҡжҡӮеӯҳпјҲж…Һз”ЁпјҒпјҒпјҒпјү
  • git pull --rebase --autostash еңЁж“ҚдҪңејҖе§Ӣд№ӢеүҚе»әз«ӢдёҖдёӘдёҙж—¶еӯҳеӮЁ

  • git stash еҪ“еүҚеҲҶж”Ҝжңүжӣҙж”№пјҢжӯӨж—¶жғіеҲҮжҚўеҲҶж”ҜпјҢдҪҶдёҚжғіжҸҗдәӨжӯЈеңЁиҝӣиЎҢзҡ„е·ҘдҪңпјҢеҸҜд»ҘеӮЁи—ҸиҝҷдәӣеҸҳжӣҙеҫҖе Ҷж ҲжҺЁйҖҒдёҖдёӘж–°зҡ„еӮЁи—Ҹ

  • git stash list иҰҒжҹҘзңӢзҺ°жңүзҡ„еӮЁи—Ҹ

  • git stash apply [stash@{0}] йҮҚж–°еә”з”Ёд№ӢеүҚзҡ„еӮЁи—ҸпјҢй»ҳи®Өдёәз¬¬дёҖдёӘ,еӮЁи—Ҹзҡ„еҶ…е®№д»Қз„¶еңЁж ҲдёҠ

  • git stash drop [stash@{0}] з§»йҷӨж ҲдёҠзҡ„еӯҳеӮЁпјҢй»ҳи®Өдёәз¬¬дёҖдёӘ
  • git remote -vжҹҘзңӢиҝңзЁӢд»“еә“
  • git remote add [shortname] [url] ж·»еҠ дёҖдёӘиҝңзЁӢд»“еә“ең°еқҖ(url)е’Ңзј©еҶҷ(shortname)git
  • git remote rename origin icode дҝ®ж”№жҹҗдёӘиҝңзЁӢд»“еә“еңЁжң¬ең°зҡ„з®Җз§°пјҢжӯӨе‘Ҫд»ӨжҢҮе°Ҷ origin иҝңзЁӢеҗҚж”№дёә code
  • git remote rm еҲ йҷӨжң¬ең°зҡ„иҝңзЁӢй“ҫжҺҘ
  • git push gitlab 1.0.3 е°Ҷжң¬ең°жҺЁеҲ° gitlab иҝңз«Ҝзҡ„ 1.0.3 еҲҶж”Ҝ
  • git pull gitlab 1.0.3 жӢүеҸ–иҝңз«Ҝзҡ„ 1.0.3 еҲҶж”Ҝ

жҒўеӨҚеҲ йҷӨиҝңз«ҜдёҚеӯҳеңЁзҡ„жң¬ең°еҲҶж”Ҝ

ж–№жі•дёҖпјҡ

  1. git log -g еҸҜжҳҫзӨәеҮәиў«иҜҜеҲ еҲҶж”Ҝзҡ„ commitIdпјҢжӯӨе‘Ҫд»ӨдјҡжҳҫзӨәеҮәеҲҮжҚўеҲҶж”Ҝзҡ„и®°еҪ•д»ҘеҸҠcommitи®°еҪ•зӯүгҖӮ
  2. git branch [branchName] [commitId] // еҹәдәҺиў«иҜҜеҲ еҲҶж”ҜжңҖж–°зҡ„ commitId ж–°е»әеҲҶж”ҜпјҢжӯӨж—¶ж–°е»әзҡ„еҲҶж”ҜдёҠдјҡеҢ…еҗ«жңҖж–° commitId д№ӢеүҚзҡ„жүҖжңү commitгҖӮ
    ж–№жі•дәҢпјҡ
  3. git reflog еҸҜжҳҫзӨәиў«еҲ йҷӨеҲҶж”Ҝзҡ„commitId
  4. git branch [branchName] [commitId] ж–°е»әеҲҶж”Ҝ
    жӯӨж–№жі•иҰҒдјҳдәҺ git log -g еӣ дёәеҰӮжһңеҲ йҷӨзҡ„еҲҶж”ҜжңҖеҗҺдёҖж¬ЎжҸҗдәӨж—¶й—ҙеӨӘд№…еҫҲйҡҫжүҫеҲ°
